## 1. The D2D interconnect physical stack and what must be modeled (micro-bump/C4/hybrid-bond, RDL, TSV, interposer; signal+power+thermal paths)

A die-to-die (D2D) link is a short, dense, vertically-and-laterally distributed interconnect that is *electrically, thermally, and mechanically* coupled. To produce physically-consistent multimodal telemetry we must model three co-located transport paths through the *same* physical structures:

- **Signal path** — the per-lane transmission channel: driver pad → micro-bump/C4/hybrid-bond → RDL trace → (optional) TSV → interposer trace → **explicit signal-return path** → receiver pad. Because UCIe/AIB/BoW are **single-ended (SE)**, the return current does not have a dedicated companion conductor; it flows through the **PDN/reference structures** (ground bumps, ground TSVs, reference plane/mesh). Signal and reference nets are therefore electromagnetically inseparable and must be co-modeled (see §2/§3).
- **Power path** — the PDN delivering V<sub>DD</sub> through (partly shared) bump fields, RDL meshes, TSVs and interposer planes; its impedance Z(f) and droop are what couple supply noise into the link as jitter. For SE signaling this same PDN/ground network is the signal return, so a PDN/return-path defect (e.g., a missing ground bump) is simultaneously a power-integrity and a signal-integrity fault.
- **Thermal path** — vertical interconnects (µbump/C4/TSV) simultaneously conduct heat; self-heating raises Cu resistivity and silicon-substrate loss, which feeds back into both signal and power paths.

## 2. Step 1 — EM extraction: geometry → RLGC / broadband S-parameters (tools, mesh/ports, frequency range, de-embedding, passivity/causality)

The first stage converts each physical element's geometry+materials into a frequency-domain electrical model. Two regimes are used and cascaded:

**(a) Quasi-static RLGC extraction** for electrically small structures (bumps, vias, short RDL):
- *Tools (commercial):* ANSYS Q3D Extractor (MoM/FEM hybrid; outputs full R[Ω], L[H], C[F], G[S] matrices; "Lumped RLGC" mode for arbitrary 3D bumps, "Equivalent TL" for parallel same-length lines). Cadence Sigrity PowerSI 3D (adaptive FEM, low-freq conditioning to **100 Hz** — essential for PI/transient correctness).
- *Open path:* closed-form analytic RLGC (Savidis & Friedman 2010 for TSV; symmetric T two-port: series R–L Cu via + skin terms, shunt Cox in series with Si MOS path Csi∥Gsi; the oxide-Si MOS capacitance is nonlinear C–V, accumulation/depletion swing ~70%). NumBDA floating-random-walk C extraction validated vs Sdevice TCAD (47× speedup). These can be re-implemented in NumPy/scikit-rf with no license.

**Single-ended return-path co-extraction (mandatory requirement).** Because the return current of every SE lane closes through the PDN/reference network, Step-1 extraction must **co-extract each signal net together with its return path** (ground bumps/ground TSVs/reference mesh/plane), not as an isolated 2-port over an ideal ground. Concretely: (i) define an explicit **reference/return assignment** per lane (which ground bumps/TSVs carry its return); (ii) extract signal+PDN nets in one multi-port model so that SE crosstalk, common-impedance (return-path) coupling, and SSO/PSIJ paths are physical; (iii) treat **return-path discontinuities** (missing/high-R ground bump or ground TSV, reference-plane gap) as **first-class injectable defects** (§6). Without co-extraction, IL, SE crosstalk, and SSO/PSIJ are not physically meaningful.

**(b) Full-wave 3D EM** for the interposer traces, TGV/TSV transitions, and any structure approaching λ/10 at Nyquist:
- *Tools:* ANSYS HFSS (CPW/TGV/micro-via to **100 GHz** channel / 20 GHz PDN, validated vs VNA); CST Microwave Studio (TSV/interposer, z(f)=2R+jωL, y(f)=G+jωC cascade; captures slow-wave mode transition ~0.5 GHz on σ=10 S/m Si); Cadence Clarity 3D (distributed FEM, multi-block synchronized interposer/D2D extraction from SPD). Coaxial-TSV+RDL+bump validated HFSS-vs-ADS **0.1–120 GHz** with S11 error 0.4%, S21 error 0.18%.

**Materials to set (corpus anchors):** Cu σ=5.8×10⁷ S/m (ρ=1.724×10⁻⁸ Ω·m @20 °C); SiO₂ εr=3.9, tanδ=0.001; Si (CMOS) σ=10 S/m, **εr≈11.7 (11.68–11.9, small dispersion to high f)**; HR-Si σ≈0.01 S/m; glass εr=5.3, tanδ=0.004; BCB εr=2.6; oxide liner 100–200 nm.

**PVT-corner re-extraction (not only T-scaling of R).** Materials above are at 20 °C; the macromodel library must be populated across the PVT grid, accounting for **all** corner-sensitive parameters, not only conductor ρ(T) and substrate-loss(T): (i) **dielectric Df(T,f) / Dk(T,f)** drift (re-extract or rescale per §7); (ii) **geometry tolerance** spread of bumps/TSVs/RDL (pitch, diameter, liner thickness, dishing) as a parametric ±range; (iii) supply-voltage corner where it changes driver R_on / pad-C operating point. Either re-extract at grid corners or ship a parameterized macromodel with documented scaling laws for each.

**Ports and de-embedding:** GSG/GSSG ports at bump pads; on-wafer measurement uses GSG-probe with G–S–G **open/short de-embedding** (NTHU) or, frequency-limit-free, the **T-matrix double-DUT cascade** T₁·T_DUT·T₂·T_DUT·T₁ = T_total inverted for T_DUT (valid to ≥40 GHz, used for µbumps/TSVs/TGV). For released datasets the open **scikit-rf IEEE 370-2020 2x-thru** de-embedding (NZC needs a 2x-thru; ZC corrects fixture impedance) to 50 GHz is the recommended reproducible path. **Crosstalk / multi-port reference handling:** state aggressor-port termination explicitly (50 Ω SE passive vs active-driver source impedance); SE crosstalk and COM/VTF crosstalk must be computed as a **power-sum over all aggressors with correct port terminations and reference renormalization**, never pairwise; renormalize the multi-port S-matrix to the actual SE reference before cascading.

**Macromodel realization (mandatory before time-domain use):** convert the broadband S-parameter table into a stable, passive, **causal** rational model via **Vector Fitting** (Gustavsen-Semlyen pole-residue, VECTFIT3) followed by **passivity enforcement** (residue-matrix-eigenvalue / Hamiltonian perturbation) **and an explicit causality step** — passivity ≠ causality. Enforce causality via a **Kramers-Kronig / Hilbert-transform consistency check** on the fitted response and explicit **delay (propagation) extraction** so the macromodel is **minimum-phase plus a pure delay** (avoids acausal pre-ringing). Without passivity *and* causality enforcement, transient (GetWave) and BER sims diverge. Validate the model bandwidth covers ≥3× Nyquist (e.g. ≥48 GHz for a 32 GT/s / 16 GHz-Nyquist UCIe lane).

**Validation anchors (measured to model):** single-TSV IL = 0.35 dB @20 GHz (HR-Si); TSV-GCPW −0.20 dB/mm @10 GHz, −0.02 dB/TSV; capacitive→inductive crossover ~2 GHz; analytic-RLGC vs 3D-solver S-params agree to 20 GHz (TSV+bump+RDL).

## 3. Step 2 — Channel assembly and D2D reference channels (UCIe Standard vs Advanced / BoW / AIB loss budgets and BER targets)

The assembled channel must reproduce the standard-specific budgets below; these are the *nominal/healthy* baselines on which defects ride. All signaling families here are **single-ended**; the listed Z₀ values are SE unless noted.

*Unit convention for BW density:* areal figures are **TB/s/mm²** (bytes); shoreline/edge figures are **Tb/s/mm** (bits). All shoreline numbers are stated **bidirectional aggregate** at the nominal bump pitch and lane count of the cited configuration (Advanced @45 µm; Standard @100–130 µm); areal density scales ≈ 1/pitch² (×3.24 from 45→25 µm). Footnote the exact pitch/lane assumption when reusing any single number, since a uni-vs-bidirectional or bit-vs-byte mix produces a spurious ~8× discrepancy.

**Compliance methodology differs by reach.** Long-reach Ethernet-style uses COM/ERL (§4). UCIe short-reach (≤2 mm) does **not** use COM — *not* because the link is "too electrically short" (COM is an SNR-with-optimal-EQ FOM and is not invalidated by channel length), but because COM's modeling assumptions do not match a UCIe PHY: (a) COM/ERL are defined for **differential, 50/100-Ω-referenced** channels, whereas UCIe is **single-ended**; (b) UCIe uses **source-series-terminated, non-50-Ω-referenced** drivers, so a 50-Ω S-parameter COM mis-states the *delivered* voltage — exactly the reason VTF folds in termination-R and pad-C; (c) UCIe is **forwarded-clock**, not the embedded-CDR/DFE that COM's EQ search assumes; and (d) at ~2 dB loss the COM EQ optimization is degenerate. Hence UCIe uses a **Voltage Transfer Function (VTF)** approach: the VTF includes Tx/Rx termination R and pad C (it is realized delivered voltage, *not* 50-Ω-referenced S21). Three deterministic checks: (1) VTF-loss curve must stay **above** the per-rate/per-package loss mask, DC→Nyquist; (2) VTF-crosstalk (power-sum of aggressor-RX-V/victim-source-V over **all** aggressors) must stay **below** the crosstalk mask; (3) a noiseless/jitterless behavioral-Tx/Rx eye-mask sim with minimum required EQ must clear a rectangular eye:

| UCIe eye mask | Eye height | Eye width | EQ requirement |
|---|---|---|---|
| 4–16 GT/s | 40 mV | 0.75 UI (no-EQ) | Tx EQ recommended @16 GT/s |
| 24–32 GT/s | 40 mV | 0.65 UI (with EQ) | Tx 1-tap de-emphasis **required** ≥24 GT/s; Rx EQ optional |

Other UCIe electrical anchors: Tx min swing ~0.4 V, recommended V_OH < 0.85 V (and ≤ RX_VCC+100 mV); pad cap Tx/Rx ~250/200 fF (Advanced), ~300 fF (Standard); T-coil ~20×20×2 µm; Rx amplifier BW ≥0.75× data rate (=1.5× Nyquist); Z₀ 50 Ω SE (100 Ω differential equivalent).

## 4. Step 3 — Driver/Rx, equalization, and IBIS-AMI; statistical-eye / COM / BER computation (Init vs GetWave; when statistical suffices for low-loss short reach)

The driver/Rx + EQ are layered onto the assembled channel via the **IBIS-AMI** interface (open, royalty-free; portable C shared libraries). Two execution modes, both needed:

**(1) Statistical / LTI — `AMI_Init`.** Assumes analog Tx, analog Rx, and passive interconnect are LTI, so the whole analog channel is one impulse response h_AC(t). The composite is the sequential convolution **h_eq(t) = h_AC(t) * h_TE(t) * h_RE(t)** (Tx-EQ and Rx-EQ as impulse responses); eye/BER derived analytically by PDF propagation. Fast, exact for static FFE/CTLE; *cannot* capture adaptation or pattern dependence.

**(2) Time-domain / non-LTI — `AMI_GetWave`.** Receives a sampled bit stream (already convolved with the channel) and returns the modified waveform plus, for the Rx, recovered `clock_times`. EQ may be non-LTI: DFE, adaptive CTLE (Mode=Adapt), CDR, runtime tap selection. Slower, statistically limited by bit count. **Correctness traps to encode:** the documented *double-counting pitfall* (Tx=Dual/Rx=Init-Only: the impulse handed to Rx Init already contains Tx-EQ); and the statistical-vs-transient *jitter discrepancy*.

**When statistical suffices:** for low-loss short-reach D2D (UCIe ~2 dB @16 GHz; AIB near-lossless; BoW short reach), the channel + static Tx FFE / Rx CTLE is effectively LTI with DFE off, so `AMI_Init` statistical eye is accurate and cheap — ideal for sweeping thousands of lanes×timesteps. Reserve `AMI_GetWave` for adaptive-EQ failure injection, CDR/forwarded-clock skew, and pattern-dependent BER bursts where defects create non-LTI behavior.

**COM / ERL for the long-reach / organic-Standard cases (differential, 50/100-Ω-referenced only)** (open `com_ieee8023_93a.m`): COM = 20·log₁₀(A_s/A_ni) dB, with A_s the FOM-maximizing-EQ signal amplitude and A_ni the noise+interference amplitude at DER0. FOM = 10·log₁₀(A_s²/Σσᵢ²) where σᵢ² = σ_ISI²+σ_J²+σ_Xtalk²+σ_TX²+σ_SpecNoise². EQ search: TX FFE c(−2),c(−1),c(1) with cursor c(0)=1−Σ|taps| (min ~0.62 NRZ); CTLE DC gains g_DC,g_DC2 (zero f_z=f_b/4); DFE solved analytically (N_DFE=14 UI NRZ / 16 UI PAM4). Rx 4th-order Bessel-Thomson, f_r=0.75·f_b; M=32 samples/UI; DER0=1e-5 (NRZ). **Pass threshold COM ≥ 3 dB.** Companion ERL = 20·log₁₀(P⁻¹(DER0)) with PTDR time-gating (Grr/Gloss). A single physics core (ISI+crosstalk PDF + jitter + Gaussian noise) co-generates COM *and* BER/bathtub consistently; documented biases: COM **underestimates** Tx jitter (ratio σ²_J,IEEE/σ²_J = 1−|ΣH_kH_{k-1}|/ΣH_k² < 1) and **overestimates** crosstalk (worst-phase vs phase-averaged), residual std-COM error ≤0.44 dB over 8 backplane configs. (COM/ERL are **differential-only**; do not apply to the SE UCIe/AIB/BoW reach — see §3.)

**Bathtub/BER mapping (dual-Dirac):** TJ(BER) = DJ(δδ) + 2·Q_BER·σ_RJ, with Q_BER = 6.4/6.7/7.0/7.3/7.6 at BER = 1e-10/-11/-12/-13/-14 (so the canonical TJ@1e-12 = DJ + 14.069·σ_RJ). This converts any extracted-channel jitter into the BER label without simulating ~10/BER bits.

**Sample-count vs BER-floor (label uncertainty for GetWave BERs).** Statistical/dual-Dirac BERs extrapolate to arbitrarily low BER, but any **transient GetWave** BER is bounded by the simulated bit count N: the lowest *directly observable* BER is ≈1/N, and an observed-zero-error run only bounds BER ≲ 3/N (95% one-sided, rule-of-three). To *measure* (not extrapolate) 1e-15 directly would need ~10¹⁵–10¹⁶ bits — infeasible — so GetWave-derived labels at the 1e-15 target are **extrapolated** (dual-Dirac fit), and each carries a quantified confidence interval from the fit; record N and the resulting label uncertainty alongside every GetWave BER so the released labels expose, rather than hide, their statistical confidence.

## 5. Step 4 — Jitter / noise / crosstalk decomposition and budgeting for parallel D2D buses

Parallel D2D buses are SSO/crosstalk-dominated and forwarded-clock, so jitter/skew (not eye/COM alone) are the primary health observables. For SE buses the SSO/return-path coupling enters through the shared PDN/ground extracted in §2. The budgeting taxonomy and the injection kernels:

**Jitter taxonomy (label schema):** 1 UI = TJ + eye-width; TJ = RJ + DJ; DJ = DDJ(ISI+DCD) + PJ + BUJ. RJ Gaussian (unbounded); DDJ data-dependent; PJ from supply/ground (PDN-droop tone); **BUJ from crosstalk**. This maps each defect class to a jitter component: resistive-open → ISI↑; aggressor lane → BUJ↑; PDN droop → PJ↑.

**Combining rule (link budget):** random (independent Gaussian) components combine by RSS of σ's; deterministic (bounded) components add linearly (peak). Total: **TJ(BER) = ΣDJ_i + 2·Q_BER·√(Σσ_i²)**.

**Injection kernels (reusable, from SerDes-Toolbox jitter model):**
- RJ: t ← t + Rj·randn (Rj = σ)
- DJ (dual-Dirac/bounded): t ← t + Dj·rand[−1,1]
- SJ (periodic): Tx t ← n·UI + Sj·sin(2π·f_Sj·n·UI)
- DCD: t ← t + DCD·(−1)ⁿ (odd/even edges)
- Rx AWGN: wave ← wave + Rx_Noise·randn (V into the SE Rx; 100 Ω diff equivalent)

**Crosstalk → BUJ (analytic, no SPICE):** victim edge time-shift = (crosstalk-pulse amplitude)/(victim edge slope); displacement of distorted vs clean crossing = ½ the crosstalk contribution; superpose all aggressors for worst-case BUJ_pp. Validation anchor: victim 2 Gb/s, aggressors 1 Gb/s, edge shift 39→41 ps over 500→700 mV swing, model-vs-measure error ~4.2 ps. BUJ scales with aggressor/victim amplitude and inversely with victim slope.

**Crosstalk fault taxonomy & coverage (MAF/MDSI):** the canonical **Maximal Aggressor Fault (MAF, Cuviello et al.) model defines 6 crosstalk faults per victim → 6N faults / 6N two-vector tests**: positive glitch g_p, negative glitch g_n, rising-delay d_r, falling-delay d_f, **rising speed-up s_r, falling speed-down s_f**. Worst-case excitation = all N−1 aggressors switch in the worst direction while the victim is held (glitch) or transitioned (delay/speed). Error thresholds: V_p_th, V_n_th, V_r_th, V_f_th, Δt_g (min glitch duration), Δt_d (min sampling margin). If the campaign intentionally omits the two speed faults (reducing to a **4N glitch/delay subset** for tractability), this must be stated explicitly as a **deliberately reduced subset — not a "complete" coverage claim**. MDSI extends to arbitrary interposer topology with explicit RLC (better for irregular RDL/branch nets). This linear (6N, or reduced 4N) count lets us enumerate a *tractable, balanced* labeled defect campaign per bundle. Quantitative MAF anchors: 1 V rail, 100 ps edges, 0.35→0.10 µm; crosstalk delay ~0.1 ns for 3 mm interconnect; glitch up to 0.8–0.9 V on 1 V rail for ~8–10 mm lines at 0.1 µm.

**Power-supply-induced jitter (couples §7 voltage → link):** the LTI kernel H_PSIJ(ω) = (PSRR_p/Slope)·PSRR′(ω)·exp(−jω·T_p0/2)·sinc(ω·T_p0/2). Per-stage measured anchors (0.35 µm I/O): pre-driver S_P ~0.7 ps/mV peaking 120–200 MHz; last-stage S_L ~0.12 ps/mV flat to ~200 MHz; worst corner SS/−40 °C ~248 ps pk-pk (12.4 %UI); shared-vs-decoupled supply penalty ~53 ps. **Superposition validity rule (a labeling boundary):** two-tone error <4.8 % when each tone <25 % V_DD (small signal → tones may be linearly summed = "nominal supply jitter"); up to 33.3 % for >25 % V_DD (large signal → injected deep-droop power-fault class, no superposition). Per-lane workflow: take the §7 droop time series → FFT → ×H_PSIJ(ω) → IFFT → per-edge TIE; the sinc/exp phase term makes PSIJ lane- and frequency-correlated (realistic correlated cross-lane jitter, not i.i.d.). *Note:* these 0.35 µm / 3.3 V / 500 Mbps PSIJ anchors are off-target and must be **rescaled** to ~0.8–1.0 V multi-GT/s UCIe (see §10), not used as drop-in.

## 6. Step 5 — Defect injection: the fault model (defect → δ-RLGC/lumped-RLC → measurable signature)

This is the core of ground-truth generation. Each defect is injected as a **perturbation of the extracted RLGC/S-parameter model** (or a lumped element added at a node), then propagated through Steps 2–4 to produce co-registered telemetry deltas and labels. The convention: **open = high series R (+ΔL for voids); short/bridge = low shunt R (+ΔC); crack = +series R + intermittent/time-varying.** Because SE lanes share a return, **return-path/ground defects** (missing or high-R ground bump/TSV) are injected as a series-R/inductance perturbation on the *return* net, raising common-impedance crosstalk and SSO/PSIJ — a class unique to SE signaling.

### 6.1 Defect taxonomy → electrical representation → parameter ranges

| Defect class | Where | Electrical model | Parameter range (corpus) |
|---|---|---|---|
| **Resistive open** | µbump, RDL via, TSV, hybrid-bond pad | series R_open (1st-order RC for high-R; **2nd-order distributed** when R_open ≈ driver R_on) | ~tens Ω (weak) → >1 MΩ / 10s GΩ (full open); 100 Ω detectable @1 GHz; 150 Ω detectable @500 kHz; worst-case delay when R_open ≈ R_on and at *intermediate* line position |

### 6.2 Quantified defect → telemetry-signature mapping

**(a) Reflectometry / SREL (TDR kernel).** ρ = (Z_L − Z₀)/(Z_L + Z₀); inverse Z_L = Z₀(1+ρ)/(1−ρ). Open → ρ=+1 (step doubles); short → ρ=−1; matched → 0. *Polarity rule:* impedance step-up (series R, series L) → same-sign reflection; step-down (shunt C, short) → opposite-sign. Reactive defects give exponential settle τ → L=τ·Z₀, C=τ/Z₀ (termination); in-line L≈½τ·Z₀, C≈½τ/Z₀. Position x = v_p·t/2; resolution Δx_min = v_p·t_r/2 (35 ps → ~3.5 mm on RG-58). Z₀=50 Ω SE / 100 Ω diff. Example: 50→75 Ω RDL step → ρ≈+0.2. **Lossy/dispersive caveats (critical on Si interposer, unlike RG-58):** (i) **multiple reflections / masking** — a near discontinuity partially masks a far one, and re-reflections add ghost features; deconvolve or gate. (ii) **dispersion/loss-induced risetime degradation** — the incident edge slows with distance on a lossy Si interposer, so spatial resolution Δx **degrades along the channel** (it is not the constant best-case RG-58 value). (iii) The integral-of-reflected-wave method is risetime-**independent only on non-dispersive lines**; on a lossy interposer it is only approximately so, so the multi-rate-augmentation trick (rendering one labeled defect at several sampling rates) is valid only within the low-dispersion band — document the band. Treat the RG-58 35 ps→3.5 mm number as a best-case reference, not the interposer resolution.

**(b) Eye-height closure (Bogatin defect→eye transfer):** a localized ±20 % Z discontinuity gives ~2 % eye-height loss at slow rates, **up to ~5 % at ≥20 Gbps**, near-linear in discontinuity length; **low-Z (40 Ω SE / 80 Ω diff) closes the eye more than high-Z (60 Ω SE / 120 Ω diff)**; degradation **worse near the Rx** (discontinuity proximity effect). Frequency domain: series-discontinuity reflections from both ends → return-loss ripple whose spacing ∝ 1/electrical-length; field "cares" about ≥30 mil features once >16 Gbps. (These are 5-inch-PCB rules-of-thumb percentages; **rescale**, do not drop in, for sub-cm interposer reach via the short-channel extraction — use as qualitative shape + sanity check only.)

**(c) Resistive-open → delay → skew:** added delay Δt ≈ 0.69·R_open·C_load (high-R, 1st-order); for low-R opens use the 2nd-order distributed model (max delay at intermediate position). This Δt becomes inter-lane **skew**. Detectability is **VDD- and test-speed-dependent**: three ranges (always/VDD-dependent/never); wire-delay-dominant paths best detected at *max* VDD, gate-delay-dominant at *low* VDD — so a low-R open may be labeled "detectable only at high VDD," co-registering link labels with the voltage modality.

**(d) Crosstalk fault separation:** a non-zero-R bridge shifts the victim's **steady-state DC level**; increased C_c adds **delay + glitch** → eye-width (jitter) and eye-height (amplitude) closure. This separability lets the benchmark distinguish "bridge" from "coupling" classes on adjacent lanes.

**(e) Eye-size vs DC-resistance divergence (empirical signature shape):** corrosion/crack experiments (PRBS7, 1.25 Gbps; BER grid 32 phases × 127 levels = 4064 points; healthy BER<1e-17) show **eye-size declines gradually/monotonically** while **DC resistance stays flat then jumps** at end-of-life (failure-time agreement <0.6 %; eye-size most diagnostic, Fisher index 8.19 > BER-avg 8.18 > eye-height 7.59 > eye-width 7.31). This is the canonical *multimodal divergence* an anomaly detector must learn: emit a flat-then-jump R channel co-registered with a continuously-degrading eye/BER channel. Skin effect makes the eye sensitive to incipient cracks *before* DC open, and smaller cracks become detectable as data-rate rises — so data-rate is a defect-sensitivity knob. (This study is solder/board-level at 1.25 Gbps, **not** µbump/TSV at tens of GT/s — use for signature *shape* and sensitivity ordering only.)

**(f) Engine choice:** LTI defects → PDA worst-case eye (EH = c₀ − Σ_{k≠0}|c_k|) + StatEye recursive PDF convolution (per-cursor double-delta at ±c_k, prob 0.5), fast, closed-form, scales to thousands of lanes. Nonlinear/pattern-dependent defects (saturating driver, adaptive-EQ failure, data-dependent droop) → AMI_GetWave transient or an LSTM surrogate (28 Gbps PAM4, LSTM>GRU>RNN, cap PRBS length to bound TBPTT error). Open back-ends: **PyBERT** (Touchstone+IBIS-AMI, eye/bathtub/Rj-Dj-ISI-DCD), **scikit-rf** (channel synthesis/cascade), **ngspice/Xyce** (transient).

## 7. Step 6 — Electro-thermal and PI-SI co-simulation: temperature/voltage effects on the channel (modality-coupling and CONFOUNDING that motivates fusion)

The four modalities are *not independent*; they share physical state, which is precisely why fusion is needed and why benign drift can masquerade as a defect.

**Temperature → conductor loss (the dominant T→S-parameter coupling):** scale ρ(T) = ρ₀[1 + α(T−20)], α_Cu = +0.00393/°C (≈+3.9 %/+10 °C, ≈+39–40 %/+100 °C). Skin depth δ(f,T)=√(ρ(T)/πfµ) (~2.1 µm @1 GHz); R_s ∝ √(ρ(T)·f); conductor attenuation **α_c(f,T) = α_c(f,T₀)·√(1+α(T−20))** → +18 % loss at +100 °C, penalizing Nyquist most. Worked anchor: 10 mΩ trace @RT → ~14 mΩ @125 °C; +0.5 dB/inch added conductor IL @150 °C (224G). Add roughness factor K_SR (Huray/Cannonball) for +3–8 % R at high f. Dielectric drift handled separately: FR-4 25→150 °C, Dk +5–10 %, Df +15–20 %; system impact at 224G/56 GHz: **eye height −40 %, eye width −25 %, COM −35 %** (suppressed to ~15 % with Df<0.001 laminate). This **Df(T,f)** drift must also feed the PVT-corner re-extraction of §2.

**Temperature → silicon-interposer substrate loss:** HR-Si (>5 kΩ·cm) substrate loss is ~0 below ~80 °C then rises sharply via thermal carrier generation → **HR-Si substrate loss 0.13 dB/mm @150 °C and 0.56 dB/mm @200 °C, both @5 GHz** (distinct mechanism from the RT TL-loss figure of <0.56 dB/mm-to-40 GHz in §1 — same numeral, different structure/condition; always tag loss figures with (structure, frequency, temperature)). LR-Si (25 Ω·cm) is more T-insensitive; there is an f<1 GHz vs f>1 GHz cross-over (loss rises vs falls with T). Implement open via Sze ρ(T) (E_G(0)=1.169 eV, α=4.9e-5 eV/K, β=655 K) + Wadell CPW closed-form — no TCAD needed. **This makes a clean hard-negative/confounder class:** die self-heating (channel 196 °C → Si-surface 153 °C) climbs substrate loss → eye closes → BER/COM worsen, *without a hard defect* — the detector must separate benign thermal drift from a true open.

**Voltage/droop → link (PDN co-analysis, open NGSPICE recipe):** build the hierarchical PDN (on-chip decap / Si-interposer+TSV / package / PCB); for SE signaling this PDN **is** the signal-return network (§2), so its impedance enters both droop and signal crosstalk. Frequency domain: AC sweep 0.1–20 GHz @100 pts/decade; Z-target = flat **35 mΩ** with **+20 dB/dec above 3.4 GHz knee**, Z_target=(V_DD·ripple)/I_ref, ripple=5 % V_DD, I_ref=0.5·I_max. Time domain: PWL-triangular di/dt sources; metric **VVI = ∫[max(V_min−V,0)+max(V−V_max,0)]dt** outside the 95–105 % V_DD band, plus violation-node count — a per-timestep voltage label and severity score. The **I/O-correlation knob (0–1.0)** sets SSO synchronization (worst-case droop at ~0.9–1.0), coupling bursty link activity to droop. SSO state model (N01/N10/N00/N11) folds *steady-state* supply droop into the statistical eye (eye-height within 2.7–3.3 % of HSPICE, BER to 1e-16; DBI forbids N01+N10>NT/2). Defect injection: degraded Z(f) (open/high-R decap, missing MIM 200–2000 pF, TSV/bump PDN-R bump, **return-path/ground-bump open**) → labeled droop anomaly.

**Multi-resonance droop spectrum (sets PJ tones for §5 PSIJ):** f=1/(2π√LC). **First droop 50–200 MHz** (L_pkg2+L_die ∥ C_die), dominant, >15 % V_DD at die; second droop (L_pkg1 ∥ C_pkg) ~5 %; third droop (L_MB ∥ C_MB) lowest. A current pattern repeating at resonance builds catastrophic droop. On-die detector realism: z14 sensor 62 µW, 2 ns latency, 0.9 % V_DD accuracy; adaptive clocking recovers ~17 % droop magnitude (relevant for forwarded-clock skew).

**The confounding feedback loop (why fusion is required):** self-heating → ρ↑ (~4 %/+10 °C) → larger IR-drop/droop → CMOS F_max/V_min/jitter degradation → more power → more heat. Vertical current concentration in shared TSV/µbump nets produces localized IR-drop + L·di/dt. Thus a hot lane shows *simultaneously* higher IL (link), higher droop (voltage), and faster aging (§8) — a single latent (temperature) drives three modalities, which a single-modality detector cannot disambiguate from a true interconnect defect.

## 8. Step 7 — Aging/drift trajectories: EM (Black) + thermomechanical (Coffin-Manson) → R(t) to evolve incipient degradation over field-time

Aging is the slow (quasi-static, multi-hour-to-month) per-lane R(t) channel that complements fast/transient defects. Three independent generative mechanisms feed a rising series-R into the §6 link model and the §7 IR-drop model.

**(a) Electromigration — Black MTTF + 3-phase R(t).** MTTF = A·j⁻ⁿ·exp(E_a/k_BT). The R(t) trajectory is *flat (nucleation) → flat (incubation) → step jump (Joule-heating positive feedback) → linear ramp (growth)*, with δR=[L_v(t)·ρ_liner·(W+2H+2t_liner)/t_liner] − ρ_Cu·W·H and void growth dL_v=J_a·Ω·dt; **fail = +10–20 % R**. Compact de Orio model: t_f = t_n + t_g, each **lognormal** (σ_c=41 MPa, L_via=0.07 µm, Z*=5.0, ρ=2.5e-8 Ω·m, j=1.33 MA/cm², E_s=0.45±0.11 eV, D_s0=6.7e-9 cm²/s; median t_n=8.5 h σ=0.38, median t_g=8.0 h σ=0.7). **Note t_f is NOT the sum of the medians** — medians of lognormals do not add linearly; report t_f as the **median of the convolved (t_n+t_g) distribution** (≈17.5 h, σ≈0.41), and never write "8.5+8.0=17.5" (the medians sum to 16.5 h; the discrepancy is the lognormal convolution, not an error). Parameter anchors by material/joint:

| Material/joint | E_a | n | j / notes |
|---|---|---|---|
| Cu (surface/interface diff.) | 0.7–0.9 eV (0.9 most cited) | 1.1–1.3 (surface); 1–2 general | on-chip j >1, approaching 10 MA/cm² |
| Al (grain-boundary) | 0.4–0.5 eV | — | — |
| SnPb µbump (FOWLP) | 1.0 eV | 2 | A=3.64e-3; **crit j ≈ 4.0–4.5×10⁴ A/cm²**; MTTF 3–7239 h; **local peak ≈ 7.17×10⁴ A/cm²** |
| Cu-Cu hybrid bond | 0.9 eV (borrowed; source lacks Ea/n) | — | I=1.5 A, ~200 °C; **strong bond 3500–5022 h vs weak ~170 h (>20× spread)**; FEA peak 3.4×10⁶ A/cm² at PSV edge |
| SnAgCuNi solder | 0.77±0.12 eV | — | EM onset j 1×10³–1×10⁴ A/cm² |

*(All solder/µbump EM current densities are ~10³–10⁴ A/cm²; on-chip Cu is ~10⁶ A/cm² (MA/cm²). State every j WITH its exponent.)* The **bond-quality latent** (void/strength) sets each lane's TTF, giving realistic heterogeneity; n encodes the mode (n≈2 nucleation-limited, n≈1 growth-limited).

**Blech length / short-length immortality (must apply to µbumps/TSVs/short RDL).** Black's law alone over-predicts failure for short segments: below the **Blech critical (j·L) product**, back-stress halts void growth and the segment is **EM-immortal**. Because µbumps, TSVs, and short RDL stubs are *short*, a large fraction may sit below (jL)_crit — apply the jL threshold per segment before assigning a finite TTF, otherwise the aging channel is systematically over-pessimistic.

**(b) Thermomechanical fatigue — Coffin-Manson + Norris-Landzberg.** N_f = D·(Δε_p)⁻ᵏ links thermal-cycling (the temperature modality) to crack initiation. 30 µm µbump anchor (one consistent (D,k) pair): **D=15.306, k=1.075 → N_f = 15.306·(Δε)⁻¹·⁰⁷⁵ ≈ 1331 cycles** at the corpus strain for −55/+125 °C (crit strain ~1.57 %; this is the value reconciled with the ~1250-cycle order-of-magnitude anchor — the earlier "D=21.9, λ=0.93" pairing was inconsistent with this N_f and is dropped). Field de-rating via Norris-Landzberg AF = (f_field/f_test)⁻ᵐ·(ΔT_field/ΔT_test)⁻ⁿ·exp[(E_a/k)(1/T_max,field − 1/T_max,test)]: SnPb n=1.9, m=1/3, E_a/k=1414 K (≈0.122 eV); SAC305 n=2.65, m=0.136, E_a/k=2185 K (≈0.188 eV).

**(c) IMC growth (slow baseline drift).** Total (Cu₆Sn₅+Cu₃Sn) IMC thickness ∝ √(anneal time), E_a≈0.57 eV (Q=56.6–88.9 kJ/mol) — a monotonic resistance/embrittlement drift distinct from cyclic fatigue and EM voiding (a third aging channel).

**(d) Population spread — distribution scope (lognormal vs Weibull).** Distinguish the two statistics by scope: **per-via / single-mechanism EM TTF is lognormal** (the de Orio t_n/t_g of (a) — use lognormal as the per-lane EM sampler). **Weibull is for series/competing weak-link populations**, i.e. at the **bundle/system level**, where a lane = N series segments and the lane TTF is the weakest-link minimum over its segments → Weibull F(t)=1−exp[−(t/η)^β] (η=63.2 % life; β>1 for wear-out, β<1 for infant mortality; µbump η>1200 cycles). Do **not** use Weibull as the per-via EM law nor lognormal as the bundle law. Thermomechanical-fatigue lane populations are likewise Weibull at bundle level. Failure event = +10–20 % R. This produces a realistic infant-mortality vs wear-out mix.

**(e) Open coupled solver.** EMSpice 2.1 integrates all of this: Q_jh=j²ρ Joule heating + external thermal map → Arrhenius D_a=D₀exp(−E_a/k_BT) → Korhonen stress PDE → void → δR(t)=(V_v−V_crit)/(WH)·[ρ_Ta/(h_Ta(2H+W)) − ρ_Cu/(HW)] → IR-drop via MNA G(t)u(t)=Pj(t) with **time-varying G(t)** (R rises as voids grow). Validated vs COMSOL: stress RMSE 3.42e7 Pa, ~1.8 % avg error, **>200× speedup (~1.6 s/tree)** — fast enough to Monte-Carlo many lanes. The Joule-heating jump couples aging *back* into the temperature modality (local hotspot at the void), producing correlated cross-modal anomalies.

**Propagation into telemetry:** rising R raises per-lane RC/IR-droop → insertion loss at Nyquist ↑, propagation delay ↑ → slow downward drift of eye height/width, gradual **lane-to-lane skew spread**, COM/BER walk-down, and (via §7 IR-drop) larger droop — all timestamped from nucleation through hard fail, giving the incipient → progressive → fail label sequence.

**Time-alignment / co-driving recipe:** (1) Define a global timebase: a fast UI/symbol grid (e.g. 31.25 ps for 32 GT/s) nested inside a slow telemetry-sample grid (µs–ms), nested inside an aging clock (h–months). (2) At each *aging* tick, EMSpice rolls R(t) for every lane (sampling per-via t_n/t_g from lognormal, applying the Blech jL immortality screen, and sampling bundle-level lane TTF from Weibull per the bond-quality latent), and feeds δR into both the channel S-params (link) and the PDN G(t) (voltage). (3) At each *telemetry* tick, 3D-ICE emits T(lane,t); ρ(T), Df(T,f), and substrate-loss(T) rescale the channel; the NGSPICE PDN (= SE return network) under the current di/dt + I/O-correlation emits V(t)/VVI; the first-droop tone (50–200 MHz) and SSO PJ feed H_PSIJ to produce correlated cross-lane TIE; PyBERT/AMI-Init recomputes eye/BER/COM-or-VTF/jitter/skew; the TDR kernel renders the SREL trace from the current Z(x). (4) The injected-defect parameters and the +10–20 % R / COM<3 dB (diff long-reach) / VTF-mask / eye-mask thresholds yield co-registered ground-truth: binary fault flag, continuous severity, defect class, and x-position — all from the *same* latent, so a droop event ⇒ jitter spike ⇒ eye/BER degradation ⇒ aging acceleration appear as one causal chain.

**Key fidelity caveats to document:**
- **LTI/statistical limits.** PDA/StatEye/COM/AMI-Init are LTI-only — invalid for saturating drivers, slew-dependent CDR, data-dependent droop, large rail collapse; those require GetWave transient, Volterra, or explicit worst-case patterns. PSIJ/PSRR kernel is small-signal (the 25 %-V_DD superposition boundary is the labeling line between nominal jitter and a power-fault class).
- **BER-label confidence.** Statistical/dual-Dirac BERs are extrapolations; any transient GetWave BER is bounded by bit count (observable floor ≈1/N, zero-error bound ≲3/N). The 1e-15 target is never directly simulated — it is dual-Dirac-extrapolated with a recorded confidence interval (§4); ship N and label uncertainty with each GetWave BER.
- **Geometry/condition mismatch (rescale, don't drop in).** PSIJ numbers are 0.35 µm/3.3 V/500 Mbps (re-scale to ~0.8–1.0 V multi-GT/s UCIe); Bogatin eye-closure % are 5-inch PCB rules-of-thumb (rescale for sub-cm interposer); the corrosion eye/BER study is solder/board-level at 1.25 Gbps, not µbump/TSV at tens of GT/s — use only for signature *shape* and sensitivity ordering. Cu-Cu hybrid-bond paper lacks E_a/n (borrow 0.9 eV from a complementary source — stated in-line in §6/§8); EMSpice 2.1 does not tabulate D₀/E_a/TCR (pull from foundry PDK or Korhonen/Black primaries).
- **SE/diff & return-path correctness.** UCIe/AIB/BoW are single-ended; SE Z₀=50 Ω (diff equiv 100 Ω). COM/ERL are **differential-only** and must not be applied to the SE reach. SE signal-return co-extraction (§2) is mandatory; a missing ground bump is a first-class injectable SI+PI defect.
- **Reliability-statistics scope.** Per-via EM TTF is lognormal; Weibull only at the series/weak-link bundle level (§8d). Apply the Blech jL short-length immortality screen so short µbumps/TSVs/RDL are not over-pessimistically aged.
- **Short-reach compliance correctness.** Do *not* apply Ethernet COM to ≤2 mm UCIe links — wrong on assumptions (SE, non-50-Ω-referenced SST drivers, forwarded-clock, degenerate-EQ low loss), which corrupts labels; use VTF + rectangular eye mask. Reserve COM/ERL for organic-Standard/long-reach differential channels.
- **TDR/SREL on lossy interposer.** The single-discontinuity ρ kernel ignores multiple-reflection masking, dispersion-induced risetime degradation (resolution worsens with distance — unlike RG-58), and the integral method's risetime-independence holds only on non-dispersive lines; gate/deconvolve and document the valid low-dispersion band for multi-rate augmentation (§6.2a).

**Open tooling gaps (and mitigations):**
- *No released open multimodal D2D dataset* exists (this benchmark fills exactly that gap). Underlying methods are open; SerDes/Signal-Integrity Toolbox, HSPICE, HFSS/Q3D/Sigrity, Silvaco ATLAS are commercial but each has an open substitute: scikit-rf + SignalIntegrity + PyBERT (channel/link), ngspice/Xyce (transient/PDN), EMSpice v1.0 BSD (EM/IR), analytic RLGC + Wadell/Sze (extraction/substrate-loss). The IBIS-AMI interface and `com_ieee8023_93a.m` are open and citable.
- *EMSpice 2.1 has no public 2.1 code* (only v1.0 BSD + published 2.1 equations) — extend v1.0 with the published δR(t)/MNA forms.
- *Full-wave EM is the only commercial dependency without a turnkey open equal at >40 GHz* — mitigate by pre-extracting a parameterized RLGC/S-param library once (offline, any solver) and shipping the Touchstone macromodels (signal+return co-extracted, PVT-cornered) with the released benchmark, so downstream regeneration is 100 % open-tool.

The net result is a DATE-quality, standards-traceable, physically-grounded generator in which every link/temperature/voltage/aging sample is produced by an open or open-reimplementable engine from a single shared latent, yielding co-registered telemetry and self-consistent ground-truth labels — and an explicit, documented map (the §10 traceability table) of where each numeric anchor is silicon-validated versus where it is a methodology template requiring rescale.
