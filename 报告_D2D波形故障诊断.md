# 基于原始波形的 Chiplet 晶粒间（D2D）互连故障智能诊断
### —— 兼论“多模态遥测的价值取决于观测的有损性”

VLSI 测试课程 Mini Research Project ｜ 方向三：D2D Link 异常检测
日期：2026-06-10

---

## 摘要

异构集成把晶粒间（die-to-die, D2D）互连变成了 chiplet 系统新的可靠性瓶颈：单封装内微凸点数以万计、节距亚 10 µm，且存在大量一次性结构测试无法预见的在役退化模式。本项目以一个**纯 ngspice、完全可复现**的 D2D 链路仿真器为数据源，构造了包含 **10 类故障 / 786 条样本**的**原始接收波形（raw RX waveform）数据集**，并在其上系统地比较了线性基线与一维卷积网络（1-D CNN）的故障诊断能力，进一步完成了三项消融实验。主要结论：

1. **诊断**：CNN 在 10 类故障上达到 **80.3%** 准确率，显著优于多项逻辑回归基线（70.2%），表明在“故障同时出现于全温度区间”的较难数据上，深度模型的非线性确有价值。
2. **严重度回归**：对两类连续漂移故障（分布式串阻 `r_drift`、并联电容 `c_drift`）做严重度回归，R² 分别达 **0.867 / 0.998**。
3. **核心洞见（本文最重要的贡献）**：温度/电压跌落（droop）是一个**良性混杂因子**——它和真实故障一样会闭合眼图。我们发现**多模态遥测（temperature/droop）的价值取决于主观测量的“有损程度”**：当模型可见完整模拟波形时，遥测几乎冗余（波形本身已编码温度/droop）；而在**真实可获得的有损观测（一个标量眼裕度）**下，遥测变得不可或缺——它把“良性高应力链路”的**误报率从 37.5% 压到 0%**。
4. **区域消融**：故障特征主要集中在**跳变沿（signal 区）**；其中阻抗不连续（反射型故障）**只能在沿区被检出**（沿区 0.94 / 平台区 0.00），印证其属于反射域、需 TDR/多模态才能覆盖。

代码、数据生成脚本与全部图表均可一键复现（见第 7 节）。

## 1. 背景与动机

### 1.1 为什么 D2D 互连成为可靠性瓶颈
先进硅的经济性推动系统从单片大芯片走向**多 chiplet 解耦 + 中介层/桥接重新集成**。单封装内的 D2D 互连数量已从 Lakefield 的 <1,000 攀升到 Ponte Vecchio 的近 100,000，I/O 密度向 10,000 IOs/mm² 演进、节距跌破 10 µm。每根互连都经过微凸点、RDL、可能还有 TSV，封装因此拥有成千上万个独立失效机会，其中相当一部分只在**部署之后**才显现——电迁移、C4 凸点热机械疲劳、NBTI/PBTI 漂移、翘曲诱发的成簇缺陷等。

### 1.2 为什么需要“在役/诊断态”健康评估，而非一次性结构测试
工业界主流答案是结构测试（IEEE 1838 晶粒包装寄存器加载确定性图案、捕获 pass/fail、用备用 lane 修复）。它严谨但本质上是**制造时一次性**活动，无法观测**弱、参数化、漂移、老化**这类在役才发展出来的失效模式——它们表现为**时序/裕度的缓慢退化**而非逻辑错误，只有持续监测眼裕度/误码/lane 偏斜的运行时监控才能捕获。

### 1.3 混杂（confounding）：单一信号天然有歧义
D2D 链路并非孤立退化：其眼裕度/误码会被**供电跌落（droop）**与**温度/热漂移**调制。一次温升或一次重负载 droop 都会**闭合眼图**，与真实缺陷的征象**重叠**。因此把“良性环境波动”与“萌生缺陷”区分开，**本质上需要融合多模态遥测**，把环境共模建模并扣除。这正是本项目的科学核心。

## 2. 问题定义

- **测试问题**：给定一条 D2D 链路在某工况下采集到的**接收端模拟波形**（诊断/后硅 bring-up 态可获得，例如片上眼图监测/示波采样），自动判别其**故障类型**（10 类，含“健康”），并对连续退化做**严重度回归**；同时回答——在**真实运行时只能拿到极有损的观测（如一个眼裕度标量）**时，加入片上温度/droop 遥测能否提升诊断、尤其能否**抑制良性高应力导致的误报**。
- **为何重要**：D2D 互连是 chiplet 良率/可用性的瓶颈（§1）；参数化/老化故障对逻辑测试不可见，必须从模拟层面的健康信号诊断；而单一信号的混杂歧义使“朴素阈值”不可靠。
- **AI 为何适合**：正常工作流形高维且与 PVT 强相关、标注的现场失效稀缺、退化渐进——这类“学正常、判偏离 + 多模态去混杂”的问题天然契合统计/ML 方法（课程推荐的 Autoencoder/LSTM/统计异常检测同源）。
- **输入/输出**：输入＝一维 RX 波形（128 UI × 8 样/UI ＝ 1024 点），可选叠加标量遥测 [温度, droop]；输出＝10 类 softmax（诊断）或连续严重度（回归）或健康/故障二分类（有损检测）。

> **观测真实性的诚实说明**：原始模拟波形属于**诊断/后硅态**信号（对应文献中 Miao 等的眼图-CNN、Usama 等的波形-AE），**并非**运行时遥测。运行时真正可得的是**有损**的眼裕度/误码计数/温度/droop 寄存器。本文据此把实验分为“完整波形”与“有损标量”两种观测体制，二者的对比正是核心洞见所在。

## 3. 数据来源与仿真方法

### 3.1 仿真器
纯 ngspice（无任何商业 EM/AMI 工具）、纯标准库 Python 封装，参数全部取自项目 SOTA 报告与仿真方法深挖文档。链路模型：

```
行为级驱动 (PWL 经 Ron 输出) → TX 微凸点(R+L+C) → N=20 段 RLGC 阶梯 → RX 微凸点 → RX 焊盘C + 轻终端
                                  ↕ 逐节点 Cc 耦合的两条相邻 aggressor lane（串扰）
```

关键物理：UCIe 级短距低摆幅单端（Vdd=0.8 V，16 GT/s → UI=62.5 ps，Z0=50 Ω），导体 R 带铜温度系数 tc1=3.9e-3/℃。

### 3.2 为支撑本研究而做的三项关键扩展
1. **瞬态电阻热噪声**：在受害 lane 各电阻上串入 `trnoise` 热电动势，RMS＝`noise_gain·√(4kTR·Δf)`。物理意义：噪声随 **√T** 随温度上升（构成混杂）、随 **√R** 随阻性故障上升（故障自带噪声指纹）；并修复了无噪声时 BER 恒为 1e-300 的伪值。
2. **温度相关驱动 Ron(T)**：`Ron(T)=Ron0·(T/T0)^1.5`（迁移率退化）。**这是温度闭合一条良好匹配短链路眼图的主导机制**——导体 R 仅 0.7 Ω，相对 45 Ω 驱动 + 80 Ω 终端可忽略，若不引入 Ron(T)，温度在波形中几乎不可观测。引入后健康眼幅 504→419 mV（27→110 ℃）。
3. **连续漂移故障 + 良性可观测 droop**：新增 `r_drift`（分布式串阻 [15,150] Ω）与 `c_drift`（并联电容膨胀），作为严重度回归目标；并加入一个**良性、可观测**的工况 droop（记入 `vdroop_frac` 遥测列，区别于 `supply_droop` 故障），作为第二个混杂因子。

### 3.3 故障类型（10 类）与数据集
| 类别 | 机制 | 主要征象 |
|---|---|---|
| healthy | — | 干净眼图（随温度/droop 闭合） |
| resistive_open | 单段大串阻 | 幅度骤降 |
| bump_void_aging | 凸点空洞渐增串阻 | 幅度下降 |
| supply_droop | 摆幅降低（故障态） | 幅度下降 |
| r_drift | 分布式串阻（连续） | 幅度梯度下降 |
| c_drift | 并联电容膨胀（连续） | 沿变缓/ISI |
| crosstalk_cap | 耦合电容增大 | 平台/沿处噪声扩散 |
| bridge_short | 对 aggressor 并联电阻 | 电平/扩散 |
| hotspot | 局部热区 + 局部退化 | 幅度下降 |
| impedance_discontinuity | 3 段 L 阶跃（反射） | **仅在跳变沿可见** |

- **样本**：786 条，每条 1024 点；故障**跨 27/85/110 ℃ 全温区**（关键：避免“高温⇒健康”的标签捷径），健康覆盖 27–110 ℃ 密集温度网格。
- **遥测列**：`temp_C`、`vdroop_frac`（良性、可观测）。
- **设计要点**：所有样本共用同一条 PRBS 发送序列（固定 `seed_v`），使故障+噪声成为唯一变化源，并使“沿区/平台区”掩膜对所有样本一致。
- **可复现性**：单条波形仿真+重采样约 1.2 s；全数据集约 16 min；公开脚本，无需任何工业数据。

数据可视化：每类眼图见 `sim/out/wave/inspect/eyes.png`（健康为清晰睁开眼，阻性/droop 类幅度坍缩，c_drift 沿变缓，crosstalk 模糊，impedance 在幅度上几乎不可见）。

## 4. 方法设计

### 4.1 两种观测体制（本文的关键变量）
- **完整波形**（诊断/后硅态）：输入 1024 点全波形。
- **有损标量**（真实运行时）：把波形压成**单一眼裕度标量**（90–10 百分位电平差），模拟片上裕度寄存器实际能给的信息。

### 4.2 模型与输入/输出
- **基线**：多项逻辑回归（线性），输入＝标准化波形（或标量），输出＝类别。
- **AI 方法**：一维 CNN（3 层卷积 + **平均池化与最大池化拼接** + 两层 MLP 头）。拼接两种池化的动机：平均池化保留**稳态电平/幅度**指纹、最大池化保留**峰值/沿**信息——仅用全局平均池化会把波形塌缩成均值、丢失幅度与极值的区分。可选把标量遥测 [温度, droop] 在全连接头处与卷积特征拼接（融合）。
- **严重度回归**：同 CNN 结构，单输出 + MSE，仅在 `r_drift`/`c_drift` 子集上训练。
- **类不平衡**：交叉熵采用逆频率类权重。
- **划分**：分层 75/25（训练 588 / 测试 198），固定随机种子。

### 4.3 三项消融
- **遥测消融**：A（仅波形）vs B（波形＋温度/droop），在**两种观测体制**下分别做，对比总体准确率、高应力子集准确率、以及**健康误报率**。
- **区域消融**：仅用**沿区** vs 仅用**平台区** vs 全波形训练 CNN（对所有样本用同一掩膜），定位“故障特征住在波形何处”；为稳健性对每种区域做 **3 次重初始化取均值±std**。

## 5. 实验结果

### 5.1 故障诊断（10 类，测试集 n=198）
| 模型 | 准确率 |
|---|---|
| 逻辑回归基线 | 0.702 |
| **1-D CNN** | **0.803** |

CNN 较线性基线 **+10 个百分点**。混淆矩阵见 `sim/out/wave/plots/confusion_cnn.png`：crosstalk、c_drift 近乎完美；阻性类之间（resistive_open / r_drift / bump_void_aging）存在可理解的混淆（都属串阻幅度故障）。

### 5.2 严重度回归（连续漂移）
| 故障 | MAE(归一化) | R² |
|---|---|---|
| r_drift（分布式串阻） | 0.107 | 0.867 |
| c_drift（并联电容） | 0.012 | **0.998** |

c_drift 严重度几乎完美可回归；r_drift 略低，因其与温度同样表现为串阻、存在轻度混杂。散点图见 `severity_*.png`。

### 5.3 遥测消融：核心洞见 —— 价值取决于观测有损性
| 观测体制 | A 仅信号 | B 信号＋遥测 | 关键变化 |
|---|---|---|---|
| **完整波形** | 总体 0.803，健康误报 0.333 | 总体 **0.864**，健康误报 0.333 | 总体 +6%，但误报未降 |
| **有损标量眼裕度** | acc 0.707，**高应力健康误报 0.375** | acc 0.732，**高应力健康误报 0.000** | **误报 37.5%→0%** |

**解读**：完整波形已隐含温度/droop（它们闭合眼图），故遥测多为冗余、仅小幅提升总体；而在真实运行时只剩一个眼裕度标量时，**良性高应力链路与真实故障在该标量上重叠**（见“关键图” `lossy_confounder.png`：灰色带内健康与故障的裕度相互重叠），单看裕度必然误报，**加入温度/droop 遥测后误报被完全消除**。这正是多模态融合在 D2D 在役监测中的**真正价值所在**，也回答了课程的“AI 何时有效/何时不必要”。

### 5.4 区域消融：故障特征住在哪里（3 次重初始化均值±std）
| 区域 | 准确率 |
|---|---|
| 全波形 | 0.806 ± 0.019 |
| 仅沿区（signal） | 0.785 ± 0.023 |
| 仅平台区（held） | 0.660 ± 0.041 |

总体上**故障信息集中在跳变沿**（沿区≈全波形≫平台区）。更有意思的是**逐类**分布（`region_ablation.png`）：

- `impedance_discontinuity`：沿区 0.94 / 平台区 **0.00** —— 反射型故障**只在沿处现形**，幅度/平台完全看不见，印证其需要 TDR/多模态覆盖。
- `healthy`：平台区 **1.00** —— 健康的干净稳态电平最具判别力。
- `r_drift`：沿区 0.78 > 平台区 0.38；`hotspot`：沿区/全波形优于平台区。

即**不同故障的判别信息分布在波形的不同部位**，这为“片上比较器最佳采样位置/多探针”的理论问题提供了可计算的经验证据。

### 5.5 案例研究（混杂去除）
取一条 **110 ℃ 的健康链路**：眼裕度 ≈ 420 mV，落入与中度故障重叠的灰色带；仅看裕度的检测器将其误判为故障。融合遥测的检测器获知“温度 110 ℃，该裕度与‘热致健康’的预期一致”，判其健康——这正是把“良性环境波动”从“萌生缺陷”中分离出来的 GLRT-with-nuisance 思路的经验体现。

### 5.6 运行时与开销
单条波形仿真+重采样约 **1.2 s**（CPU）；CNN 在 588 样本上训练约 1 分钟（CPU），推理为毫秒级。整套实验在普通笔记本上数分钟内可复现。

## 6. 技术分析与 Insight

### 6.1 AI 为什么有效
- 在“故障跨全温区”的较难数据上，类间边界非线性，CNN 较线性基线 **+10%**；其 avg+max 池化同时抓住幅度与沿/峰值特征。
- 连续退化（c_drift）在波形中是**单调、可分辨**的形变，回归 R² 高达 0.998。
- 多模态融合在**信息瓶颈**处（有损标量观测）发挥作用：它提供了波形已丢失的环境共模信息，从而把混杂歧义解开。

### 6.2 最重要的洞见：遥测的价值是“观测条件性的”
朴素直觉是“多加遥测总是更好”。本文给出**反例与边界**：当主观测足够丰富（完整波形）时，温度/droop 已被隐式编码、遥测近乎冗余；只有当主观测被压成运行时真实可得的**有损标量**时，混杂才制造出不可分的重叠，遥测才成为**消除误报的关键**（37.5%→0%）。这把“多模态融合是否有用”从一个口号变成一个**有判决条件的科学结论**，也直接服务于课程“AI 在测试中的局限性”的要求。

### 6.3 AI / 本方法的局限性
1. **匹配良好的短链路对参数/热扰动不敏感**：导体 R 远小于驱动+终端，故温度、分布式串阻、热点、阻抗阶跃在“原始眼幅”上长期不可见——必须显式引入 Ron(T)、绝对分布式串阻等机制才使其可检（这本身是一条物理洞见，也提示**单一眼图度量天生迟钝**）。
2. **阻抗不连续在眼图中几乎不可见**（平台区 0.00），属反射域故障，需 TDR/反射系数模态——印证多模态/多探针的必要性。
3. **原始波形并非运行时遥测**：本文据此区分两种观测体制并诚实标注；运行时的真实形态是有损标量/误码序列。
4. **r_drift 与温度混杂**：分布式串阻与温度同样抬升串阻，单波形难以完全分离其严重度（R² 0.867 < c_drift 0.998），需遥测辅助。
5. **数据规模与单次划分**：786 样本、单次分层划分（区域消融已用 3 次重初始化稳健化）；类别不平衡用类权重缓解；全为仿真数据，缺乏现场标定。
6. **CNN 相对线性基线优势有限**：在更易的数据版本上二者几乎持平，符合课程“不卷模型、看是否真解决问题”的导向——本工作的贡献在**问题重构与洞见**，而非新模型。

## 7. 结论与复现

### 7.1 结论
本项目交付了一个**开放、可复现、物理接地**的 D2D 互连原始波形故障诊断基准与方法：CNN 10 类诊断 80.3%、连续漂移严重度回归 R² 0.87/0.998；并给出本文核心贡献——**多模态遥测的价值取决于观测的有损性**：在真实有损观测下，融合温度/droop 把良性高应力误报从 37.5% 压到 0%；同时通过区域消融定位了各类故障的判别信息所在（沿区为主，阻抗不连续仅见于沿区）。这构成对“在役 D2D 链路健康多模态融合”空白的一个**问题重构 + 可证伪演示**式贡献，而非新 ML 模型。

### 7.2 未来工作

### 7.3 复现说明
```bash
cd sim
# 1) 生成原始波形数据集（约 16 min）
python3 -c "from d2dsim.wave_campaign import run_wave_campaign; run_wave_campaign(out_dir='out/wave')"
# 2) 可视化（眼图/波形/遥测覆盖）
~/miniconda3/envs/drl_hw2/bin/python inspect_waves.py --data out/wave
# 3) 训练 + 全部消融 + 出图
~/miniconda3/envs/drl_hw2/bin/python ml_diagnose.py --data out/wave
```
- 仿真器：纯 ngspice + 标准库（`d2dsim/`）；ML：`drl_hw2` 环境（numpy/torch/matplotlib）。
- 关键产物：`out/wave/waveforms.csv`、`out/wave/results.json`、`out/wave/plots/*.png`、`out/wave/inspect/*.png`。
